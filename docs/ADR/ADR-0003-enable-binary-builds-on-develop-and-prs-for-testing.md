# ADR-0003: develop ブランチおよびプルリクエストにおけるリリース前バイナリビルドの有効化

- **Date**: 2026-05-24
- **Status**: Accepted
- **Deciders**: ayato-labs (Human), Antigravity (Agent)

## Context

これまで、Windows 用ネイティブバイナリ（`Ripen.exe` などの `.exe`）の PyInstaller によるビルドとリリースは、`main` ブランチでのマージ（セマンティックリリースによる新タグ発行時）にのみ限定されてトリガーされていました。

この運用には以下の課題がありました：

1. **リリース前テストの困難さ**: `develop` ブランチでの開発フェーズにおいて、実際にビルドされた `.exe` ファイルが正しく動作するかをリリース前にローカル環境等で検証することが難しく、リリース後に問題が発覚するリスクがありました。
2. **ビルドエラーの検知遅れ**: 依存ライブラリの追加やコード変更によって PyInstaller のビルドプロセス自体が壊れた場合、それを検知できるのが `main` への統合（＝本番リリース）のタイミングになってしまい、リリースパイプラインが停止する原因となっていました。

## Decision

`develop` ブランチへのプッシュ時、およびすべてのプルリクエスト（PR）時にもバイナリビルドを実行するよう CI/CD ワークフローを拡張します。

- `.github/workflows/main.yml` の `build_assets` ジョブの実行条件 (`if`) を緩和し、`main` に加え `develop` ブランチへのプッシュ、およびプルリクエスト時にもビルドが走るようにします。
- `main` ブランチ以外の非リリースビルド時は、リリースアセットへの登録をスキップし、代わりに GitHub Actions のアーティファクト機能（`actions/upload-artifact`）を利用してビルドされたバイナリ（`Ripen.exe`, `RipenInit.exe`）を保存（保存期間: 7日間）します。
- 開発者は、GitHub Actions の実行完了画面（Artifacts）からビルドされた `.exe` を直接ダウンロードして動作確認を行うことができます。

## Consequences

### Positive
- **テストの容易性と品質向上**: リリース前に本番同様のビルドバイナリを用いてローカルテストができるようになります。
- **エラーの早期発見**: PyInstaller のビルド失敗や依存関係の競合を、PR 段階で即座に検知できます。

### Negative / Risks
- **CI/CD ランナー消費量の増加**: プルリクエストや `develop` へのプッシュのたびに Windows ランナーによる PyInstaller ビルドが走るため、GitHub Actions の実行時間（およびコスト）が増加します。しかし、ビルド対象から `RipenInstaller` を削除したことでビルド時間が短縮されており、テストの容易性向上がこの追加コストを上回ると判断します。
