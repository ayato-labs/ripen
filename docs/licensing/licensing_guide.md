# Ripen Licensing & Activation Guide

Ripenは、オープンソース（AGPL-3.0）の透明性と、商用利用におけるLocal-Firstな信頼性を両立させるためのハイブリッド・ライセンスモデルを採用しています。

## 1. ライセンスモデルの概要

- **Personal/Community Edition**: AGPL-3.0ライセンス。個人利用や非営利目的、またはAGPLの制約を許容するプロジェクトに適用されます。
- **Commercial Edition (Professional/Enterprise)**: プロフェッショナルなチーム向けの商用ライセンス。AGPLのコピーレフト制約を受けず、エンタープライズ向けの追加機能（高度なナレッジ同期、監査ログ等）を提供します。

## 2. 技術アーキテクチャ

### 基盤: Keygen.sh
ライセンス管理には [Keygen.sh](https://keygen.sh/) を採用しています。中央サーバーでライセンスを集中管理しつつ、クライアント側での高度なオフライン検証を可能にします。

### 署名スキーム: Ed25519 & HTTP Signatures
Ripenは、数学的に偽装不可能な署名検証を行っています。
- **スキーム**: `ED25519_SIGN`
- **検証方式**: HTTP Signature (Draft standard)
- **Root of Trust**: `config.py` にハードコードされたEd25519公開鍵。

検証フロー:
1. **アクティベーション**: サーバーから `Date`, `Digest`, `Keygen-Signature` ヘッダーを含むレスポンスを受信。
2. **正規化**: 受信したヘッダーとボディを連結し、Signing Stringを再構築。
3. **検証**: 公開鍵を用いて署名の整合性を確認。
4. **永続化**: 署名済みのコンテキストをローカル（`~/.ripen/license.cache`）に保存。

## 3. マシン識別（フィンガープリント）

ライセンスの不正な共有を防ぐため、ハードウェア固有の識別子を使用しています。
- **生成ロジック**: OSのUUIDまたはMACアドレスをベースにしたハッシュ。
- **Privacy Policy**: 収集されるのは匿名化されたハッシュのみであり、個人の特定やプライバシーの侵害は行いません。

## 4. 運用・トラブルシューティング

### アクティベーション失敗時のチェックリスト
1. **ネットワーク接続**: 初回アクティベーション時は `api.keygen.sh` へのHTTPS通信が必要です。
2. **プロキシ設定**: 企業環境で通信が遮断されていないか確認してください。
3. **マシンIDの変更**: マザーボードの交換や仮想マシンのクローンによりフィンガープリントが変わった場合、ダッシュボードからライセンスをリセットする必要があります。

### オフライン運用
- 一度アクティベートされたRipenは、ライセンスの有効期限内であればインターネット接続なしで動作し続けます。
- 再検証が必要なタイミング（デフォルト: 24時間ごと）でバックグラウンドでの疎通確認を試みますが、失敗しても即座に機能が停止することはありません。
