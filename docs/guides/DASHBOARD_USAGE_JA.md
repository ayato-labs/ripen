# 透明性ダッシュボード利用ガイド (Transparency Dashboard)

Ripen のダッシュボードは、AI エージェントが何を記憶し、どこで意見が食い違っているのかをリアルタイムに可視化し、人間が最終的な知識の制御（ガバナンス）を行うためのツールです。

## アクセス方法

サーバーが起動している状態で、ブラウザから以下の URL にアクセスしてください。

- **URL**: `http://localhost:8377/history`
- **ポート**: デフォルトは `8377` です（起動オプション `--port` で変更可能）。

## 主要機能

### 1. Conflict Center (競合解決センター)
AI が既存の知識と矛盾する新しい情報を検知した際、その情報は即座に反映されず、ここに「保留中」としてリストアップされます。

- **Existing (既存知識)**: 現在データベースに保存されている確定済みの情報。
- **Proposed (提案された知識)**: エージェントが新しく保存しようとした矛盾する情報。
- **操作**:
    - **Approve & Merge**: 提案された情報を「正」として承認し、主記憶に統合します。
    - **Reject**: 提案された情報を「誤り」として却下し、主記憶を現状維持します。

> [!IMPORTANT]
> このプロセス（Human-in-the-Loop）により、誤った情報やハルシネーションによる知識ベースの汚染を防ぎます。

### 2. Activity Timeline (アクティビティ・タイムライン)
エージェントがいつ、どのようなアクション（Save/Update 等）を行ったかの全履歴を表示します。

- **Action**: 行われた操作の種類（ENTITY_SAVE, OBS_SAVE 等）。
- **ID**: 操作対象の ID または CID。
- **Agent**: 誰が（Cursor, Claude, Gemini 等）その知識を提供したか。

### 3. System Insights & Stats
システムの稼働状況や、知識の蓄積状況に関するサマリーを確認できます。

## チームでの運用シナリオ

1.  **知識の信頼性確認**: 重要なプロジェクト方針が変わった際、AI がそれを矛盾として検知したら、チームリーダーがダッシュボードで内容を確認して承認します。
2.  **トレーサビリティの確保**: 記憶がいつの間にか書き換わっていた場合、タイムラインを遡ることで「どのエージェントがいつ変更したか」を特定できます。
3.  **AI のデバッグ**: AI が期待通りに記憶していない場合、ダッシュボードを見て保存プロセスが正常に動いているか、矛盾としてブロックされていないかを確認できます。

## 注意事項
- **セキュリティ**: 現在、ダッシュボードはローカル環境からのアクセスを想定しています。公開ネットワークに晒す場合は、リバースプロキシ等での認証設定を推奨します。
- **リフレッシュ**: 5秒ごとに自動更新されますが、即座に反映させたい場合はブラウザの更新（F5）を行ってください。
